# 创建软链接
## 背景
有时候在安装某些软件的时候，组件确实在`Linux`平台上安装了，但是没有安装在默认路径下，所以导致某些命令找不到的情况，如下所示：
```sh
[root@localhost harbor]# docker-compose up -d
-bash: docker-compose: command not found
```
在安装`Docker`的时候指定了安装子组件`docker-compose`，安装路径如下所示

可以看到：`docker-compose`组件被安装在`/usr/libexec/docker/cli-plugins/docker-compose`路径下面，如果我们想要执行`docker-compose up -d`来重新部署`Docker`容器的话，则需要为`docker-compose`在默认路径下创建一个软链接。
## 创建软链接
创建软链接的命令如下所示：
```sh
ln -s /usr/libexec/docker/cli-plugins/docker-compose /usr/bin/docker-compose
```
